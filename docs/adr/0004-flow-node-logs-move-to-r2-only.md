# Flow node-level logs move from D1 to R2-only, with a D1 counter table for live badges

`flow`'s per-node enter/exit event log (powering the flow editor's Analytics tab — per-node badges and a "which entities passed through this node" drill-down) previously lived only in D1's `flow_log` table, keyed `user_id NOT NULL`. This meant content-domain flows (added later) had no equivalent at all — the code explicitly skipped node-level logging for them, since extending `flow_log` to a second, differently-keyed domain wasn't practical without either widening a NOT NULL column or forking the table.

Rather than bolt a `content_id` variant onto the existing D1 table, we moved the *detail* event log entirely to R2 Data Catalog (`flow_log`/`content_flow_log`, Iceberg, shared multi-tenant tables) for both domains, and introduced a separate, precomputed per-tenant D1 counter table (`flow_counts`/`content_flow_counts`) purely to keep the live badges cheap. The counter is recomputed every minute by re-aggregating the *entire* history of the R2 table and overwriting each active tenant's counts — deliberately not incremental, so no redelivery-safe dedup logic is needed anywhere in the write path (Cloudflare Queue delivery is at-least-once). The drill-down endpoint queries R2 directly and synchronously on click, following the precedent already set by `link`'s Users list (a direct HTTP call to Cloudflare's R2 SQL endpoint, not the `analytics` module's async queue+container pattern) — cross-table JOIN to `uniscrm.user`/`uniscrm.content` for names is untested in this codebase, with a two-step D1 lookup as the fallback if it doesn't work.

The pre-existing production R2 pipeline for flow logs was abandoned rather than migrated — nothing had ever queried it, so there was nothing to preserve. The old D1 `flow_log` table is dropped outright (not merely stopped-at) — an explicit choice to reclaim its per-tenant storage rather than leave it as unread dead weight. A direct consequence: badges reset to zero for every existing flow on rollout, since there is no backfill path from either the abandoned R2 pipeline or the dropped D1 table.
